# Инструкция по развертыванию Django-проекта на хостинге Jino

Это пошаговое руководство поможет вам развернуть ваш проект `bigz` на хостинге Jino.

## Шаг 1: Настройка в панели управления Jino

Перед загрузкой проекта необходимо настроить окружение на хостинге.

1.  **Войдите в [панель управления Jino](https://cp.jino.ru/).**
2.  **Проверьте услуги:** Убедитесь, что у вас подключены услуги **"Поддержка веб-приложений"** и **"Поддержка SSH"**. Это можно проверить в разделе "Услуги".
3.  **Настройте домен:**
    *   Перейдите в раздел "Хостинг" -> "Домены".
    *   Добавьте ваш домен `bigz.monster` и, если планируете использовать, `www.bigz.monster`.
    *   Убедитесь, что домен привязан к сервису "Хостинг".
4.  **Выберите версию Python:**
    *   Перейдите в "Хостинг" -> "Управление" -> "Настройки веб-сервера".
    *   В списке доменов найдите `bigz.monster` и выберите для него последнюю доступную версию Python (например, **3.10** или **3.11**). Это очень важно, так как проект требует Python 3.10+.
5.  **Получите данные для SSH:**
    *   Перейдите в "Хостинг" -> "Управление" -> "Доступ по SSH".
    *   Здесь вы найдете имя хоста, логин и порт для подключения. Пароль совпадает с паролем от вашего аккаунта Jino.

## Шаг 2: Загрузка проекта на сервер

Мы будем использовать `git` для загрузки проекта. Это самый удобный способ для последующих обновлений.

1.  **Подключитесь к серверу по SSH.** Используйте терминал (на macOS/Linux) или программу PuTTY (на Windows).
    ```bash
    # Замените username на ваш логин из панели Jino
    ssh username@username.jino.ru
    ```
2.  **Клонируйте репозиторий:**
    *   Сначала перейдите в директорию `~/domains`.
        ```bash
        cd ~/domains
        ```
    *   Теперь клонируйте проект с GitHub (или другого сервиса, где он хранится). Вам нужно будет указать URL вашего репозитория.
        ```bash
        # Замените URL на актуальный URL вашего репозитория
        git clone https://github.com/ваш-логин/ваш-репозиторий.git bigz.monster
        ```
    *   Эта команда скачает все файлы проекта в папку `~/domains/bigz.monster`, как и рекомендует документация Jino.

## Шаг 3: Настройка на сервере

Теперь настроим сам проект и окружение для него.

1.  **Перейдите в папку проекта:**
    ```bash
    cd ~/domains/bigz.monster
    ```
2.  **Создайте и настройте `.env` файл:**
    *   Скопируйте пример файла:
        ```bash
        cp .env.example .env
        ```
    *   Откройте файл `.env` в текстовом редакторе (`nano` или `vim`) и заполните его вашими реальными данными.
        ```bash
        nano .env
        ```
    *   Вам нужно установить `SECRET_KEY` (сгенерируйте новый, например, [здесь](https://djecrety.ir/)), `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_IDS`.
    *   Чтобы сохранить изменения в `nano`, нажмите `Ctrl+X`, затем `Y` и `Enter`.

3.  **Создайте виртуальное окружение:**
    *   Используйте команду из документации Jino, указав вашу версию Python (например, 3.10). Папка с окружением будет называться `venv`.
        ```bash
        # Убедитесь, что вы находитесь в папке ~/domains/bigz.monster
        # Эта команда использует утилиту virtualenv от Python 3.7 для создания
        # окружения с интерпретатором Python 3.10. Это стандартный способ на Jino.
        # Замените python310 на нужную версию, если она отличается.
        /opt/alt/python37/bin/virtualenv --python /opt/alt/python310/bin/python3 venv
        ```
4.  **Активируйте окружение и установите зависимости:**
    ```bash
    source venv/bin/activate
    pip install -r requirements.txt
    ```
    *   Теперь вы работаете внутри виртуального окружения (в терминале появится префикс `(venv)`).

5.  **Выполните миграции базы данных:**
    *   Так как мы используем SQLite, Django просто создаст файл базы данных.
    ```bash
    python manage.py migrate
    ```

6.  **Соберите статические файлы:**
    ```bash
    python manage.py collectstatic --noinput
    ```
    *   Эта команда соберет все CSS, JS и изображения в папку `staticfiles`.

7.  **Переместите статику для веб-сервера:**
    *   Jino ожидает статические файлы в папке `public_html`. Создадим ее и переместим туда собранные файлы.
    ```bash
    # Создаем папку
    mkdir public_html
    # Перемещаем все из `staticfiles` в `public_html`
    mv staticfiles/* public_html/
    ```

## Шаг 4: Создание файла-запускателя (passenger_wsgi.py)

Это главный файл, который Apache (`mod_passenger`) будет использовать для запуска вашего приложения.

1.  **Создайте файл `passenger_wsgi.py`:**
    *   Убедитесь, что вы находитесь в папке `~/domains/bigz.monster`.
    *   Создайте файл:
        ```bash
        nano passenger_wsgi.py
        ```
2.  **Вставьте в него следующий код:**
    ```python
    import sys, os

    # Путь к интерпретатору Python в вашем виртуальном окружении
    INTERP = os.path.expanduser("~/domains/bigz.monster/venv/bin/python")
    if sys.executable != INTERP:
        os.execl(INTERP, INTERP, *sys.argv)

    # Путь к корневой папке вашего Django-проекта
    sys.path.insert(0, os.path.dirname(__file__))

    # Загружаем переменные окружения из .env файла
    from dotenv import load_dotenv
    load_dotenv(os.path.join(os.path.dirname(__file__), '.env'))

    # Указываем Django на файл настроек
    os.environ['DJANGO_SETTINGS_MODULE'] = 'bigz.settings'

    # Запускаем приложение
    from bigz.wsgi import application
    ```
3.  **Сохраните файл** (`Ctrl+X`, `Y`, `Enter`).

## Шаг 5: Финальный запуск

1.  **Перезагрузите веб-сервер:**
    *   Вернитесь в панель управления Jino.
    *   Перейдите в "Хостинг" -> "Управление" -> "Настройки веб-сервера".
    *   Нажмите кнопку "Перезагрузить веб-сервер".

2.  **Проверка:**
    *   Откройте в браузере ваш сайт `https://bigz.monster`. Он должен заработать.

Если вы видите ошибку 500, проверьте лог-файлы в панели Jino: "Хостинг" -> "Статистика" -> "Лог-файлы ошибок". Это поможет понять, в чем проблема.
